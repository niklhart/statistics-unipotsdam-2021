Data types

This tutorial covers the fundamental data types in R:

  • Vectors
  • Factors
  • Matrices
  • Data frames
  • Lists
  • Functions

Vectors

A vector contains one or several values of a particular type. The most important types are:

  • numeric (e.g., 1.41434)
  • integer (e.g., 5 or -2)
  • logical (TRUE or FALSE)
  • character (e.g., "tutorial", "peanut", …)

In addition, each type has a special value NA, indicating missing values.

Vectors of a certain length can be allocated (initialized) as follows:

numeric(4)
## [1] 0 0 0 0
integer(4)
## [1] 0 0 0 0
logical(3)
## [1] FALSE FALSE FALSE
character(6)
## [1] "" "" "" "" "" ""

Vectors can be interconverted between the different formats:

as.character(c(2.34,1.42,0))
## [1] "2.34" "1.42" "0"
as.logical(c(2.34,1.42,0))
## [1]  TRUE  TRUE FALSE
as.numeric(c(TRUE,FALSE))
## [1] 1 0
as.character(c(TRUE,FALSE))
## [1] "TRUE"  "FALSE"

In most cases, the distinction between numeric and integer types is not very important, since they are almost always automatically converted to the required format.

Values/vectors can be combined using the command c:

c(1, numeric(2), 2.3)
## [1] 1.0 0.0 0.0 2.3
c("a", "b", "c")
## [1] "a" "b" "c"
c(TRUE, TRUE, FALSE)
## [1]  TRUE  TRUE FALSE

Regular sequences of numeric vectors can be created in different ways, too:

  1. the : command:
1:4
## [1] 1 2 3 4
2.5:4.5
## [1] 2.5 3.5 4.5
  1. function seq():
seq(from = 1, to = 7, by = 2)
## [1] 1 3 5 7

Vectors can also be named. Then, each element of the vector has its own name:

c(1,2)     # unnamed values
## [1] 1 2
c(a=1,b=2) # values named "a" and "b"
## a b 
## 1 2

Factors

The data type “factor” is designed for categorical variables, i.e. variables with a pre-defined set of values (usually, the number of categories is much smaller than the length of the vector).

x <- factor(c("small","medium","small","big"))
x
## [1] small  medium small  big   
## Levels: big medium small

The possible values are called “levels”.

levels(x)
## [1] "big"    "medium" "small"

It is also possible for a factor to have a level which is not used in the vector itself (for example, this allows to document that a certain category was not observed). In this case, define the levels as an input argument to factor:

y <- factor(c("small","medium","small"), levels = c("small","medium","big"))
y
## [1] small  medium small 
## Levels: small medium big

Factors can be unordered (the default) or ordered – corresponding to nominal and ordinal data types. For example, variable y above could be encoded as ordered. In this case, always use argument levels, and in increasing order:

y <- factor(c("small","medium","small"), levels = c("small","medium","big"), ordered = TRUE)
y
## [1] small  medium small 
## Levels: small < medium < big

Ordered factors can be compared using comparison operators (see Tutorial “Basic syntax”), quantiles can be determined, and many plotting functions respect the ordering during display.

For efficiency, factors are implemented on top of integer vectors with the levels as display values. This has important consequences for manipulation of factors because many functions trigger a conversion of a factor to its underlying type “integer”, which is generally not what you want. For example, concatenation first converts y to integer, then to character, to produce a vector consistent with the second argument:

c(y, "small")
## [1] "1"     "2"     "1"     "small"

Matrices

Matrices can be created using function matrix(). Arguments nrow/ncol specify the size of the matrix, and one of them can be left blank.

matrix(1:6, nrow = 3)
##      [,1] [,2]
## [1,]    1    4
## [2,]    2    5
## [3,]    3    6

For matrices, the operator * denotes element-wise multiplication. For matrix multiplication (which is what you often want), use %*% instead of *:

M <- matrix(1:4, nrow = 2)
M
##      [,1] [,2]
## [1,]    1    3
## [2,]    2    4
x <- matrix(1:2, nrow = 2)
x
##      [,1]
## [1,]    1
## [2,]    2
M %*% x
##      [,1]
## [1,]    7
## [2,]   10

Invertible (square) matrices can be inverted using function solve:

solve(M)
##      [,1] [,2]
## [1,]   -2  1.5
## [2,]    1 -0.5

Several vectors/matrices can also be combined by row or column using functions rbind and cbind:

rbind(1:3,4:6)  # arguments interpreted as row vectors
##      [,1] [,2] [,3]
## [1,]    1    2    3
## [2,]    4    5    6
cbind(1:3,4:6)  # arguments interpreted as column vectors
##      [,1] [,2]
## [1,]    1    4
## [2,]    2    5
## [3,]    3    6
cbind(M,M)      # returns matrix (M M)
##      [,1] [,2] [,3] [,4]
## [1,]    1    3    1    3
## [2,]    2    4    2    4
                # returns matrix (M)
rbind(M,M)      #                (M)
##      [,1] [,2]
## [1,]    1    3
## [2,]    2    4
## [3,]    1    3
## [4,]    2    4

Data frames

Data frames are designed to hold data. It is a tabular format, similar to a matrix, but where different columns can have different types (e.g. numeric or character). In many cases, each row corresponds to a different observation/record, whereas each column corresponds to one observable quantity.

Data frames can be defined as follows:

t  <- c(1, 2, 3)
id  <- c("a", "b", "c")
df <- data.frame(time = t, ids = id)
df
##   time ids
## 1    1   a
## 2    2   b
## 3    3   c

For unnamed arguments, data.frame() uses the argument names as column names:

R  <- c(1, 2, 3)
df <- data.frame(R)
df
##   R
## 1 1
## 2 2
## 3 3

Most of the time, data frames will be read from a tabular data file (e.g., a .csv file) – see Tutorial “Data import/export”. Often, data frames need further post-processing like subsetting. This is covered in Tutorial “Manipulating data frames”.

Lists

A list is like a vector, but it allows to hold heterogeneous data. Like vectors, lists can be unnamed or named, which depends on whether argument names are provided to the list function (most often, named lists are used):

list(1:3,
     "A",
     data.frame(r=1:3)
)
## [[1]]
## [1] 1 2 3
## 
## [[2]]
## [1] "A"
## 
## [[3]]
##   r
## 1 1
## 2 2
## 3 3
list(x = 1:3,
     y = "A",
     z = data.frame(r=1:3)
)
## $x
## [1] 1 2 3
## 
## $y
## [1] "A"
## 
## $z
##   r
## 1 1
## 2 2
## 3 3

Lists can also be nested:

list(x = 1:3,
     y = list(z = 1)
)
## $x
## [1] 1 2 3
## 
## $y
## $y$z
## [1] 1

Functions

Here a very short introduction to functions is given. Further details about functions are provided in the tutorials “Function calls” and “Defining a function”.

Each function consist of three parts:

  • the formal arguments (including argument names and optionally default values)
  • the function body (the actual function content)
  • an environment in which it was defined (see Tutorial “Defining a function”)

All R functions return exactly one value. More complex output must be provided in a suitable data format. Often, this is a list.

R features first-class functions. This means that functions are a data type as any other one:

  • functions can be passed as arguments to other functions
  • functions can be stored in lists
  • functions can be modified (advanced)

Typing a function name at the console reveals the content of a function. For almost all base R functions, the output is extremely difficult to read. Here’s an easy example, function force, which just evaluates its input.

force
## function (x) 
## x
## <bytecode: 0x7fd489924bf8>
## <environment: namespace:base>

Here, x is the single formal argument, x is also the function body, and the function environment is the (namespace of the) “base” package.
